## 🏗️ Architecture

```
SONiC Switch
  └── rsyslog (message-type log, UDP 514)
        └──► Zabbix host machine :514
                └── rsyslog → ./sonic-syslog/sonic-syslog.log
                        └──► Zabbix Host Docker - zabbix-agent2 (active mode)
                                └──► zabbix-server → Dashboard
```

---

## 1️⃣ SONiC Switch - Enterprise SONiC Configuration

```bash
sonic# configure
sonic(config)# logging server <ZABBIX_HOST_IP> message-type log protocol udp remote-port 514 severity notice
sonic(config)# exit

# Verify
sonic# show logging servers
```

> ⚠️ SONiC only supports one logging server entry per host address

---

## 2️⃣ Zabbix Server Syslog Configuration

### 2.1 rsyslog — Receive Syslog

Create `/etc/rsyslog.d/zabbix-sonic.conf`:

```ini
module(load="imudp")
input(type="imudp" port="514")

# Write all incoming syslog to file
action(type="omfile" file="/opt/datahub/sonic-syslog/sonic-syslog.log")
```

Set up the log file:

```bash
mkdir -p /opt/datahub/sonic-syslog
touch /opt/datahub/sonic-syslog/sonic-syslog.log
chown syslog:syslog /opt/datahub/sonic-syslog/
chown syslog:syslog /opt/datahub/sonic-syslog/sonic-syslog.log
systemctl restart rsyslog

# Verify listening
ss -ulnp | grep 514
```

### 2.2 docker-compose.yml — Add zabbix-agent2

Add under `services:`:

```yaml
  zabbix-agent2:
    image: zabbix/zabbix-agent2:latest
    container_name: zabbix-agent2
    restart: always
    environment:
      ZBX_SERVER_HOST: zabbix-server
      ZBX_HOSTNAME: <YOUR_HOSTNAME>
      ZBX_ACTIVE_ALLOW: "true"
    volumes:
      - ./sonic-syslog:/var/log/sonic-syslog:ro
    networks:
      - datahub-net
    depends_on:
      - zabbix-server
```

```bash
docker compose up -d zabbix-agent2

# Verify
docker ps | grep zabbix-agent2
```

## 3️⃣ Zabbix UI Configuration

### 3.1 Create Host

`Data collection → Hosts → Create host`

| Field | Value |
|-------|-------|
| Host name | `<YOUR_HOSTNAME>` (must match `ZBX_HOSTNAME`) |
| Host groups | `SONiC` (create new) |
| Interfaces | Type: `Zabbix Agent`, IP: `<agent2 container IP>`, Port: `10050` |

> Get agent2 container IP: `docker inspect zabbix-agent2 | grep IPAddress`

### 3.2 Create Item

`Data collection → Hosts → <hostname> → Items → Create item`

| Field | Value |
|-------|-------|
| Name | `SONiC Syslog` |
| Type | `Zabbix agent (active)` |
| Key | `log[/var/log/sonic-syslog/sonic-syslog.log,,,,skip]` |
| Type of information | `Log` |
| Update interval | `30s` |

## 4️⃣ Dashboard Setup

`Monitoring → Dashboard → Edit dashboard → Add page`

Add a **History** widget:

| Field | Value |
|-------|-------|
| Type | `History` |
| Items | Select `SONiC Syslog` |
| Show lines | `50–100` |

## 🔍 Troubleshooting

```bash
# Test rsyslog reception (run on SONiC)
sudo logger -p notice "[EVENT] test manual event from sonic"

# Watch log file on host machine
tail -f /opt/datahub/sonic-syslog/sonic-syslog.log

# Check agent2 connection status
docker logs zabbix-agent2 | grep -i "active checks\|error"

# Capture packets to verify SONiC is sending
tcpdump -i any udp port 514 -n
```

---

## 📝 Notes

- SONiC syslog message format: `<timestamp> <hostname> <SEVERITY> <process>: <message>`
- `[EVENT]` and `[ALARM]` tags are generated by the SONiC `eventd` process; interface state changes and other general logs do **not** carry these tags
- rsyslog runs as the `syslog` user — ensure log files have correct ownership
- zabbix-agent2 must run in **active** mode; item type must be set to `Zabbix agent (active)`
